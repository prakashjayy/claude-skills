# Config Subskill

Extracts all hyperparameters from the paper and produces `configs/default.yaml`.

---

## Phase 1 — Scan paper for hyperparameters

Look in: implementation details section, appendix, caption of training tables, Algorithm pseudocode.

Collect every number that controls training or architecture. Group into these buckets:

**model**: architecture dimensions, depth, number of heads, patch size, output dimension, any module-specific constants

**dataset**: root path, image size, normalization stats, split names, number of classes

**training**: epochs, batch size, optimizer (type, lr, weight decay, betas), scheduler (type, warmup epochs, min lr), gradient clipping, fp16, checkpoint directory, save frequency

**augmentation**: crop scale, color jitter strength, blur params, multi-crop sizes if applicable

**loss**: temperature values, loss component weights, any buffers (e.g. centering momentum)

**evaluation**: protocols list, k for k-NN, paper's reported baseline numbers

---

## Phase 2 — Write `configs/default.yaml`

Every value must have a comment linking to where in the paper it came from.
Use `# [Section X]`, `# [Table Y]`, or `# [not in paper — estimated]` for defaults not explicitly stated.

Template:

```yaml
# configs/default.yaml
# Hyperparameters for: <PaperTitle>
# Paper: <arxiv link>
# Generated by /paper-to-code

model:
  # Architecture: <backbone name>
  embed_dim: 384          # [Table 1] ViT-S hidden dimension
  depth: 12               # [Table 1] number of transformer blocks
  num_heads: 6            # [Table 1]
  patch_size: 16          # [Table 1] patch size in pixels
  img_size: 224           # [Section 3.2]
  out_dim: 65536          # [Section 3] projection head output dimension
  mlp_ratio: 4.0          # [not in paper — standard ViT default]
  drop_path_rate: 0.1     # [not in paper — estimated]

dataset:
  name: imagenette        # chosen by user in dataset subskill
  root: data/imagenette
  num_classes: 10
  image_size: 224
  mean: [0.485, 0.456, 0.406]   # ImageNet normalization
  std:  [0.229, 0.224, 0.225]
  train_split: train
  val_split: val

training:
  epochs: 300             # [Section 3.2]
  batch_size: 64          # scaled down from paper's 1024 for single GPU
  # paper used: batch_size=1024, 16 GPUs → effective 1024 total
  # scale lr accordingly: lr = 0.0005 × batch_size / 256

  num_workers: 4          # [not in paper — system default]
  fp16: false             # [not in paper — optional speedup]
  grad_clip: null         # [not in paper — no clipping mentioned]
  checkpoint_dir: checkpoints
  save_every: 50          # save checkpoint every N epochs

  optimizer:
    type: adamw           # [Section 3.2]
    lr: 0.000125          # scaled: 0.0005 × 64/256 = 0.000125
    lr_paper: 0.0005      # [Section 3.2] paper's base lr for bs=256
    weight_decay: 0.04    # [Section 3.2], cosine schedule 0.04→0.4 ignored for simplicity
    beta1: 0.9            # [not in paper — AdamW default]
    beta2: 0.999          # [not in paper — AdamW default]

  scheduler:
    type: cosine          # [Section 3.2]
    warmup_epochs: 10     # [Section 3.2]
    min_lr: 1.0e-6        # [not in paper — estimated]

augmentation:
  # Global crops [Section 3.2 / BYOL augmentations]
  global_scale: [0.32, 1.0]     # [Appendix D]
  global_size: 224
  num_global_crops: 2

  # Local crops (multi-crop)
  local_scale: [0.05, 0.32]     # [Appendix D]
  local_size: 96
  num_local_crops: 6            # [Section 3.2] 8 total crops = 2 global + 6 local

  # Color augmentations [BYOL / Section 3.2]
  color_jitter_strength: 0.5
  gaussian_blur_prob: [1.0, 0.1]     # [global1, global2] prob of blur
  solarize_prob: [0.0, 0.2]          # [global1, global2]

loss:
  type: dino              # or: cross_entropy, info_nce, elbo, etc.
  student_temp: 0.1       # [Section 3.1] τ_s
  teacher_temp_start: 0.04   # [Section 3.2] linear warmup
  teacher_temp_end: 0.07     # [Section 3.2]
  teacher_temp_warmup_epochs: 30  # [Section 3.2]
  center_momentum: 0.9    # [Section 3.1 / Eq. 4]

evaluation:
  protocols:
    - linear_probe        # [Section 4.1]
    - knn                 # [Section 4.1]
  knn_k: 20              # [Section 3.2] "20 NN is consistently working the best"
  knn_temperature: 0.07  # [standard — not paper-specific]
  # Paper's reported numbers (for comparison printing)
  paper_baselines:
    linear_probe: 0.770   # [Table 2] ViT-S/16, 300 epochs
    knn: 0.745            # [Table 2]
```

---

## Rules for the config file

1. **Every value has a comment** — either a paper reference or `# [not in paper — <reason>]`
2. **Keep paper's original value visible** — when you scale batch size down, keep the paper's value in a comment above
3. **Reproducibility note at top** — single comment block explaining what was changed from the paper setup
4. **No magic numbers in code** — every constant in `model.py`, `losses.py`, `trainer.py` must trace to this file
5. **Evaluate section always includes `paper_baselines`** — even if user uses a different dataset (they'll be clearly below paper numbers and that's expected)

## After writing configs/default.yaml

Print a summary of any significant divergences from the paper:

```
Config written: configs/default.yaml

Notable differences from paper setup:
  - batch_size: 64 (paper: 1024) → lr scaled accordingly
  - epochs: 300 (paper: 800 for best results) → reduce for faster iteration
  - dataset: imagenette (paper: ImageNet-1k) → expect lower absolute accuracy
  - multi-crop: kept as-is, but local crops at 96×96 may OOM at batch_size > 64

Expected training time: ~4 hours/100 epochs on a single 16GB GPU
```

Estimate training time as: `(epochs × images_per_epoch) / (throughput_img_per_sec)`. Use 200 img/s as a rough single-GPU estimate for ViT-S/16 at batch_size=64.
